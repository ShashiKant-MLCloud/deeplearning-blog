Step 01 of 12
Image → Matrix of Numbers
A CNN never "sees" an image the way you do. It receives a grid of integers — one number per pixel — each between 0 (black) and 255 (white). Everything else builds on this foundation.
Representation
Imagegrayscale = H × W matrix, values ∈ [0, 255]
Imagecolor = H × W × 3 tensor (R, G, B channels)
First step before feeding into CNN: normalize → divide every value by 255 → range becomes [0.0, 1.0]

Normalized (÷255)
Why normalize to [0,1]? Gradient descent works best when inputs are small. Raw pixel values (0–255) would cause gradients to be 255× larger than necessary, making training unstable. After ÷255 everything is in the same small range.
Color image = 3 separate matrices
Tensor shape: 5 × 5 × 3 = 75 numbers total. A real 224×224 color image = 224 × 224 × 3 = 150,528 numbers.
Step 02 of 12
The Filter (Kernel)
A filter is a small matrix of numbers — the CNN's "pattern detector". Its values are learned during training. Different values detect different patterns.
Filter notation
W = K × K × C tensor of learned weights
where K = filter size (usually 3), C = input channels
A first-layer filter on a grayscale image: 3×3×1 = 9 weights. On color: 3×3×3 = 27 weights. Each weight starts random and is refined by training.

Step 03 of 12
One Convolution Step
Place the filter over a patch of the image. Multiply every pair of matching numbers. Sum all products. Add bias. That's one output value.
Dot product + bias
output = Σr,s W[r,s] × X[i+r, j+s] + b
W = filter weights · X = image patch · b = bias · (r,s) iterates over all K×K filter positions

Step 04 of 12
Sliding Across → Full Feature Map
The filter doesn't just apply once — it slides to every valid position. Each position produces one output value. The full collection is the feature map.
Output size formula
Output size = ⌊(N + 2P − K) / S⌋ + 1
N = input size · K = filter size · P = padding · S = stride

Step 05 of 12
ReLU Activation
After convolution, every value passes through ReLU — the simplest possible function. It's just: keep positives, zero out negatives. Yet without it, stacking layers would be meaningless.
ReLU (Rectified Linear Unit)
ReLU(x) = max(0, x)
5 → 5  ·  12.3 → 12.3  ·  −3 → 0  ·  −8.7 → 0  ·  0 → 0

Why is non-linearity essential?
Without ReLU: W₂(W₁x) = (W₂W₁)x — two linear layers collapse into one. No matter how many layers you add, the whole network behaves like a single matrix multiplication. It can only learn straight-line decision boundaries.

With ReLU: each layer can "bend" the space. The network can learn curved, complex boundaries — like the shape of a face or the outline of a car.
Step 06 of 12
Max Pooling
After ReLU, max pooling compresses the feature map. It scans a small window and keeps only the largest value — throwing away exact position, preserving signal strength.
2×2 Max Pool, stride 2
Output[i,j] = max( X[2i,2j], X[2i,2j+1], X[2i+1,2j], X[2i+1,2j+1] )
Zero learnable parameters. Output size = ⌊Input size / 2⌋ for each spatial dimension.

Why "max" specifically? A high activation = "I found this feature here." The max says "this feature appeared somewhere in this 2×2 neighborhood." We don't care where exactly — just that it was there. Average would dilute a strong detection. Min would do the opposite of what we want.
Step 07 of 12
Multiple Filters → Feature Maps
One layer doesn't use one filter — it uses N filters simultaneously. Each filter scans the entire input and produces its own feature map. The N maps are stacked to form the 3D output tensor.
Layer output tensor
Output shape = H′ × W′ × N
Parameters = (K × K × C_in + 1) × N
H′,W′ = spatial output dims · N = number of filters · C_in = input channels · +1 = bias per filter

Step 08 of 12
Deep Filters — Layer 2 Onward
Layer 2 doesn't receive the original image — it receives the 32 feature maps from layer 1. So its filters are 3D blocks: 3×3×32. They look across all feature maps simultaneously to detect combinations.
3D Convolution output at position (i,j), filter n
Output[i,j,n] = Σc Σr,s W_n[r,s,c] × X[i+r, j+s, c] + b_n
c iterates over all input channels · r,s iterate over the K×K spatial filter · W_n is the 3D filter for output channel n
Why "all channels at once" matters — corner detector example
Layer 1 map A: vertical edges
High = vertical edge found
+
Layer 1 map B: horizontal edges
High = horizontal edge found
→
Layer 2 output: corner map
High = both edges present = corner!
The Layer 2 filter learns large weights for both channel A and channel B, near-zero for everything else. The only place both are high at the same position is the corner. That's what makes it a corner detector.
Step 09 of 12
Flatten → Fully Connected Layer
After all the conv + pool layers, the 3D feature tensor is unrolled into a 1D vector. Then every value connects to every output neuron through a learned weight.
Fully connected operation
z_j = Σi Wij × x_i + b_j
output_j = ReLU(z_j)
x_i = flattened input values · W_ij = weight from input i to output j · b_j = bias for output j

Step 10 of 12
Softmax — Scores to Probabilities
The last FC layer outputs raw scores (logits) — one per class. Softmax converts these into proper probabilities that are positive and sum to exactly 1.
Softmax function
P(class i) = ez_i / Σj ez_j
z_i = raw logit score for class i · e = Euler's number ≈ 2.718 · The denominator ensures all probabilities sum to 1
Drag logit scores — watch probabilities update live
Softmax probabilities
Why exponential? Why not just normalize directly?
Direct normalization: z/Σz doesn't work if any z is negative.
Exponential e^z is always positive, and amplifies differences — a score of 4 vs 2 gives much more than 2× the probability. This makes the winning class dominate clearly, which is what we want for confident predictions.
Step 11 of 12
Loss Function — Measuring "How Wrong"
The loss function takes the network's output probabilities and the correct label, and returns one number measuring how bad the prediction was. Training tries to make this number as small as possible.
Cross-Entropy Loss
L = −log( P(correct class) )
If the network assigns 99% to the right answer: L = −log(0.99) ≈ 0.01 (tiny loss, good prediction)
If it assigns 1% to the right answer: L = −log(0.01) ≈ 4.6 (huge loss, terrible prediction)

Step 12 of 12
Backprop & Gradient Descent
Once we have the loss, we compute how much each weight contributed to the error (the gradient), then nudge every weight in the direction that reduces the loss.
Weight update rule (Gradient Descent)
W_new = W_old − α × ∂L/∂W
α (learning rate) = step size · ∂L/∂W (gradient) = "how much does the loss change when W increases slightly?" · computed via chain rule (backprop)
Interactive gradient descent — 1D loss landscape

Chain rule (backpropagation): ∂L/∂W_conv = (∂L/∂output) × (∂output/∂W_conv). The second term equals the input patch that was multiplied by W — data we already have from the forward pass. Backprop is just this chain rule applied recursively from the loss all the way back to the first layer.
