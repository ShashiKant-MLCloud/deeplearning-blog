01 — INTRODUCTION

What is an RNN and why does it exist?

The motivation behind recurrent networks — why ordinary neural networks fall short for sequences.

A Recurrent Neural Network (RNN) is a class of neural network designed to process sequential data — data where order matters and where understanding context from prior steps improves prediction of future steps.

Language is the clearest example. To understand the word "bank" in a sentence, you need to know whether the previous words were about rivers or finance. To generate the pronoun "she", you need to remember that the subject introduced three sentences ago was feminine. A standard feedforward network cannot do this — it processes each input in complete isolation, with no memory of what came before.

Core Idea

An RNN solves this by maintaining a hidden state h(t) — a vector that acts as a running memory of everything the network has processed up to time step t. At each step, it updates this memory using both the new input and the old memory, then passes the updated state forward to the next step.

Where sequential context matters

🗣️
Language
Word meaning depends on prior words. Grammar depends on sentence structure built over many tokens.
🎵
Music & Audio
The next note depends on the melody so far. Speech recognition needs context across phonemes.
📈
Time Series
Stock prices, weather, sensor data — each reading depends on recent history.
🎬
Video
Understanding a frame requires knowing what happened in prior frames.
🧬
Biology
DNA sequences, protein folding — position and neighbourhood context both matter.
🤖
Robotics
Actions depend on the history of movements, sensor readings, and state.
Historical Note

RNNs were introduced in the 1980s and became dominant for sequence tasks in the 2000s–2010s. They were largely superseded by Transformers in 2017 for language tasks, but remain widely used in time-series forecasting, speech recognition, and embedded systems where memory efficiency matters.

02 — COMPARISON

ANN / CNN vs RNN — the fundamental difference

Understanding the two axes — depth vs time — and why they are fundamentally different.

This is the most important conceptual distinction. ANN and RNN both use layers and weights — but they operate on completely different axes.

Aspect ANN / CNN RNN
Primary axis Depth (layers) Time (steps)
Input processed Entire input at once — no order One token/frame at a time — order matters
Weight structure Each layer has its own unique weights W₁ ≠ W₂ ≠ W₃ Same single weight set W reused at every time step
Memory None — stateless, each sample independent Hidden state h(t) carries memory across time
Input length Fixed — must be same size every time Variable — can handle any sequence length T
Information flow Layer 1 → Layer 2 → Layer 3 (one sample) Step t → Step t+1 → Step t+2 (across time)

The analogy that makes it clear

Think of an ANN as reading a photograph — all pixels visible at once, processed layer by layer to extract features. Think of an RNN as listening to a sentence read aloud — one word at a time, each word processed in the context of everything heard so far.

ANN — data flows downward through depth (one sample, processed once): Input → [Layer 1, W₁] → [Layer 2, W₂] → [Layer 3, W₃] → Output W₁ ≠ W₂ ≠ W₃ (different weights per layer) RNN — data flows sideways through time (sequence, processed step by step): x(1) → [h(1), W] → x(2) → [h(2), W] → x(3) → [h(3), W] → Output ↑ same W ↑ ↑ same W ↑ ↑ same W ↑ h(t) passed forward at every step — memory accumulates
The Sharpest Distinction

In ANN, output of layer 1 feeds layer 2 within the same data sample. In RNN, output of step 1 feeds step 2 across different points in time. ANN goes deep. RNN goes through time.

03 — ARCHITECTURE

RNN architecture — one layer per time step

The folded and unrolled views, and what actually lives inside one RNN cell.

At its simplest, an RNN has exactly one hidden layer. This is the introductory case. At each time step, this single layer runs once — receiving the current input x(t) and the previous hidden state h(t-1), and producing a new hidden state h(t).

The folded view — one box with a loop

The compact way to draw an RNN is a single box with an arrow looping back into itself. That loop is not connecting to a different layer — it is the same layer, feeding its own output back as input for the next run. The loop is time passing.

┌────────────────────────────────────┐ │ h(t-1) loops back │ ▼ │ x(t) → [ ONE LAYER: tanh(W·x + U·h + b) ] → h(t) ─┘ │ y(t) (output / prediction)

The unrolled view — time stretched out

When you "unroll" an RNN across a sequence of length T, you draw the same box T times side by side. This makes the time dimension visible. Crucially, all T boxes share the same weights — there is only ever one layer, one W, one U, one b.

x(1) → [h(1)] ──U──▶ x(2) → [h(2)] ──U──▶ x(3) → [h(3)] → y(3) W,U,b W,U,b W,U,b ↑─────────────── same weights everywhere ──────────────↑ amber arrows = h(t) passing memory through time

Inside one time step — the neuron equation

Two things come into the layer at every step: the current input x(t) and the previous memory h(t-1). Each is multiplied by its own weight matrix, added together with a bias, then squished through tanh.

RNN hidden state update — the core equation
h(t) = tanh( W · x(t) + U · h(t-1) + b ) Where: W = input weight matrix (how to process the new word/token) U = recurrent weight matrix (how to use past memory) x(t) = current input at step t h(t-1) = hidden state from previous step (memory) b = bias vector Compare to a plain ANN neuron: h = tanh( W · x + b ) ← only ONE input The only addition: U · h(t-1). That one extra term IS the entire memory mechanism.
Key Insight — W vs U

W (input weight) asks: "What is important about this new piece of information?" — it processes x(t). U (recurrent weight) asks: "What should I carry forward from what I already know?" — it processes h(t-1). Two separate matrices because new input and past memory are fundamentally different types of information.

04 — DEEP RNN

Deep (stacked) RNN — multiple hidden layers

Why one layer is not always enough, and how stacking works across two axes simultaneously.

Just as deep ANNs learn progressively more abstract features by stacking layers, deep RNNs stack multiple recurrent layers on top of each other at each time step. This is how real-world systems are built — Google's translation system used 8 stacked LSTM layers.

A deep RNN operates on two axes simultaneously:

↑ Vertical (ANN-style depth)
At each time step, layer 1 feeds layer 2 feeds layer 3. Each layer builds richer, more abstract features from the one below. Different weights per layer: W₁ ≠ W₂ ≠ W₃.
→ Horizontal (RNN-style time)
Each layer has its own independent memory stream flowing through time. Layer 1 passes h1(t) to h1(t+1). Layer 2 passes h2(t) to h2(t+1). Same weights across time: W1 is always W1 at every t.
Layer 3: h3(1) ──U3──▶ h3(2) ──U3──▶ h3(3) → output ↑ ↑ ↑ W3,b3 W3,b3 W3,b3 │ (vertical) │ │ Layer 2: h2(1) ──U2──▶ h2(2) ──U2──▶ h2(3) ↑ ↑ ↑ W2,b2 W2,b2 W2,b2 │ │ │ Layer 1: h1(1) ──U1──▶ h1(2) ──U1──▶ h1(3) ↑ ↑ ↑ x(1),W1 x(2),W1 x(3),W1 Horizontal arrows = memory streams (different colour per layer = independent) Vertical arrows = depth (different W at each layer)

What each layer learns

LayerAbstractsMemory of
Layer 1Raw token features, word shapes, phonemesIndividual tokens flowing through time
Layer 2Grammar patterns, phrase structureGrammatical context across the sentence
Layer 3Topic, sentiment, semantic meaningHigh-level meaning of the text so far
The Two Weight Rules

Rule 1 (across time): Same layer always uses the same weights — W1 at t=1, t=100, t=1000 is identical.
Rule 2 (across layers): Different layers have different weights — W1 ≠ W2 ≠ W3.
A 3-layer deep RNN has exactly 3 weight sets {W1,U1,b1}, {W2,U2,b2}, {W3,U3,b3} — each reused infinitely across time.

05 — MATHEMATICS

The complete forward pass mathematics

Every equation from input to prediction — with shapes and intuitions.

Step 1 — Hidden state update

Equation 1 — the recurrent update (runs T times)
h(t) = tanh( W · x(t) + U · h(t-1) + b ) Initialization: h(0) = 0 (zero vector before sequence begins)

Step 2 — Output projection

Equation 2 — project hidden state to vocabulary space
y(t) = W_y · h(t) + b_y y(t) are raw scores (logits) — one number per class in vocabulary V

Step 3 — Softmax (convert to probabilities)

Equation 3 — softmax normalization
p(k) = e^(y_k) / Σⱼ e^(y_j) for each class k ∈ {1, ..., V} Properties: all p(k) > 0, Σ p(k) = 1 (valid probability distribution) Example with h(t) = [2, 1]: e^2 = 7.389, e^1 = 2.718, Σ = 10.107 p(0) = 7.389 / 10.107 = 0.731 (73.1% — correct class) p(1) = 2.718 / 10.107 = 0.269 (26.9% — wrong class)

Step 4 — Cross-entropy loss

Equation 4 — cross-entropy loss per step
L(t) = −log( p(correct class) ) Total loss over sequence: L = Σ(t=1 to T) L(t) Intuition: p = 0.99 → L = 0.010 (confident and correct — tiny loss) p = 0.73 → L = 0.313 (our example above) p = 0.50 → L = 0.693 (uncertain) p = 0.01 → L = 4.605 (totally wrong — massive penalty)

Step 5 — Softmax gradient (the elegant result)

Equation 5 — gradient of loss w.r.t. logits
∂L/∂y_k = p(k) − 1{k is correct class} For correct class: ∂L/∂y₀ = 0.731 − 1 = −0.269 (push y₀ UP) For wrong class: ∂L/∂y₁ = 0.269 − 0 = +0.269 (push y₁ DOWN) This residual (predicted − truth) flows back through W_y into the RNN.
06 — SEQUENCE LENGTH

Sequence length — how T is determined

What counts as one time step, and how to handle variable-length sequences in practice.

T (sequence length) is not fixed by nature — you define what one time step means. A step can be a character, a word, a subword token, an audio frame, or a video frame. T is then how many of those units are in your input.

Four strategies for choosing T

  • 1
    Natural boundary — end at sentence punctuation T = sentence length, varies per sample (5 words → T=5, 20 words → T=20). Hidden state reset to zero between sentences. Most intuitive for NLP — translation, sentiment analysis, NER.
  • 2
    Fixed window — always T = constant (e.g. 35) Regardless of sentence boundaries, slice text into fixed chunks. Hidden state carried across chunks. Used in language modelling and Truncated BPTT.
  • 3
    Max length + padding — for GPU batching Find longest sequence in batch, pad shorter ones with [PAD] tokens. A mask (1=real, 0=pad) tells the network to ignore padded positions. Necessary for efficient GPU matrix operations.
  • 4
    Full document — T = all tokens Rare with vanilla RNN due to vanishing gradients. LSTM can handle a few hundred steps. Speech recognition may process 100s of audio frames per second.
Practical Defaults

Sentiment analysis: T = 10–50 words. Machine translation: T = 20–100 words. Language modelling (Truncated BPTT): T = 35–100. Speech recognition: T = 100–1000 frames. Beyond ~150 steps, even LSTM struggles — Transformers are the modern solution for very long contexts.

07 — TRAINING

Training — forward pass, loss, BPTT

How RNN weights are initialized, updated, and shared across time.

The training loop — 7 steps

  • 1
    Initialise all weights randomly W, U, b ← small random values (e.g. Xavier/Glorot initialization). h(0) ← zero vector.
  • 2
    Forward pass — run all T time steps with current weights (frozen) Feed x(t1): compute h(1) = tanh(W·x(1) + U·h(0) + b). Feed x(t2): same W and U, but now h(1) flows in as memory → h(2). Feed x(t3): same W and U, h(2) flows in → h(3) → output y(3). Weights do NOT change between t1, t2, t3 within one pass.
  • 3
    Compute loss L at the end of the sequence Apply softmax → cross-entropy loss. The loss reflects errors across ALL time steps.
  • 4
    BPTT — backpropagation through time Propagate gradient backwards: t=T → t=T-1 → ... → t=1. Accumulate ∂L/∂W and ∂L/∂U from every time step.
  • 5
    Update all weights at once W ← W − η·∂L/∂W. U ← U − η·∂L/∂U. One update per full sequence.
  • 6
    Repeat for next training sentence Same weights (now updated) applied fresh to new sequence.
  • 7
    Repeat for many epochs After thousands of sentences across many epochs, weights converge. Final weights applied identically at every time step during inference.
Critical Point

Weights are frozen during the forward pass — they do not change between x(t1), x(t2), x(t3) within one sequence. Only after computing the loss for the whole sequence does backpropagation update all weights together. This is necessary because the mistake at t=T is partly caused by decisions made at t=1 — all time steps must be seen before assigning blame.

BPTT gradient equation

Backpropagation through time — gradient of loss w.r.t. h(t)
∂L/∂h(t) = ∂L/∂h(T) × ∏(k=t+1 to T) ∂h(k)/∂h(k-1) Each factor: ∂h(k)/∂h(k-1) = tanh'(z_k) × W Where tanh'(z) = 1 − tanh²(z) ← always between 0 and 1 So: ∂L/∂h(t) = ∂L/∂h(T) × ∏(k=t+1 to T) [ tanh'(z_k) × W ] ↑ This product is the problem.
08 — THE CORE PROBLEM

Vanishing gradient — why RNNs forget

The mathematical reason RNNs cannot learn long-range dependencies, shown with concrete numbers.

After defining sequence length T and completing the forward pass, the backward pass must propagate gradients from the loss at t=T all the way back to t=1. At each step backwards, the gradient is multiplied by tanh'(z) × W. Both factors are typically less than 1. Multiplying many numbers less than 1 together drives the product toward zero — exponentially fast.

The Problem in One Line

If tanh'(z) × W ≈ 0.4 at each step, then after 6 steps: 0.4⁶ ≈ 0.004. After 50 steps: 0.4⁵⁰ ≈ 10⁻²⁰. The gradient is effectively zero — the network learns nothing from early time steps.

Concrete example — T = 6, tracking "France"

Sentence: "I grew up in France and speak fluent ___"
To predict "French", the network must remember "France" from t=4. But the gradient arriving at t=4 has already passed through t=6 and t=5 twice.

Assume: W = 0.8,  tanh'(z) ≈ 0.5 (typical)
Each backward step multiplies gradient by: 0.5 × 0.8 = 0.4

t=6 (Loss):  gradient = 1.000    ← starting signal
t=5:         gradient = 1.000 × 0.4 = 0.400
t=4 France:  gradient = 0.400 × 0.4 = 0.160  ← "France" gets 16%
t=3:         gradient = 0.160 × 0.4 = 0.064
t=2:         gradient = 0.064 × 0.4 = 0.026
t=1 "I":     gradient = 0.026 × 0.4 = 0.010  ← "I" gets 1%

Weight update ΔW = learning_rate × gradient:
  t=6: ΔW = 0.01000   ← learns strongly
  t=4: ΔW = 0.00160   ← barely learns
  t=1: ΔW = 0.00010   ← effectively no learning

With T=50 or T=100, t=1 receives effectively zero gradient. The network cannot learn long-range dependencies — it is fundamentally limited to a memory span of ~10 steps.

Why this happens — the root cause

Gradients flow backward through the same tanh and the same W repeatedly. Unlike a deep ANN where each layer has different weights (so the product can be balanced), an RNN applies the same transformation at every step. One bad factor (tanh'×W slightly less than 1) gets raised to the power of T. At T=50, even 0.9⁵⁰ = 0.005. The multiplication compounds the shrinkage mercilessly.

09 — THE SOLUTION

LSTM — Long Short-Term Memory

How Hochreiter & Schmidhuber (1997) solved the vanishing gradient problem with a memory highway and three learned gates.

LSTM replaces the single hidden state of a vanilla RNN with two parallel memory systems and three gates — learned mechanisms that control what to remember, what to write, and what to reveal.

Two memory vectors instead of one

VectorNameRoleAnalogy
C(t)Cell StateLong-term memory — stable, travels across many steps with minimal distortionFiling cabinet on a conveyor belt
h(t)Hidden StateShort-term working memory — filtered, task-specific view of C(t)Sticky note on your desk

The three gates

The Gate Principle

Every gate is a sigmoid layer followed by element-wise multiplication. Sigmoid outputs 0 to 1 — acting as a valve. 0 = completely blocked. 1 = fully open. The network learns when to open and close each gate.

🗑️ Forget Gate — what to erase

f(t) = σ( W_f · [h(t-1); x(t)] + b_f ) Output: values between 0 and 1 per memory slot f(t) = 0 → completely erase that slot in C(t-1) f(t) = 1 → keep that slot in C(t-1) unchanged Example: Reading "nurse" after "chef" — forget gate outputs ≈ 0 for the gender slot → masculine gender of "chef" erased.

📥 Input Gate + Gate Gate — what to write

i(t) = σ( W_i · [h(t-1); x(t)] + b_i ) ← how much to write (0 to 1) g(t) = tanh( W_g · [h(t-1); x(t)] + b_g ) ← what to write (-1 to +1) i(t) controls the valve (sigmoid). g(t) creates the actual content (tanh — can be negative or positive). Together: i(t) ⊙ g(t) = new content to write, scaled by importance.

🛣️ Cell State Update — the memory highway

C(t) = f(t) ⊙ C(t-1) + i(t) ⊙ g(t) f(t) ⊙ C(t-1) ← erase part of old memory (scale by forget gate) i(t) ⊙ g(t) ← write part of new content KEY: This is an ADDITION (+), not a multiplication chain. The gradient of C(t) w.r.t. C(t-1) = f(t) — a learned value ≈ 1. No repeated tanh squashing. Gradient flows back freely.

📤 Output Gate — what to reveal

o(t) = σ( W_o · [h(t-1); x(t)] + b_o ) h(t) = o(t) ⊙ tanh( C(t) ) The cell state C(t) contains everything. The output gate selects which parts are relevant RIGHT NOW for the next prediction.

Why LSTM solves vanishing gradient

Gradient through the cell state — no multiplication chain
∂L/∂C(t) = ∂L/∂h(t) · o(t) · (1 − tanh²(C(t))) + ∂C(next) The ∂C(next) term is ADDED — not multiplied through tanh repeatedly. Gradient back k steps: ∂L/∂C(t) ≈ ∂L/∂C(T) × ∏ f(k) If the network learns f(k) ≈ 1.0 for important memory: 1.0^50 = 1.00 ← gradient arrives at t=1 completely intact! 0.9^50 = 0.005 ← some decay, but manageable 0.4^50 ≈ 0.000 ← vanilla RNN fate (tanh chain)

All 6 LSTM equations — complete reference

LSTM forward pass — complete
z(t) = [h(t-1); x(t)] concat inputs (shared by all gates) f(t) = σ( W_f · z(t) + b_f ) FORGET gate i(t) = σ( W_i · z(t) + b_i ) INPUT gate g(t) = tanh( W_g · z(t) + b_g ) GATE gate (candidate content) C(t) = f(t) ⊙ C(t-1) + i(t) ⊙ g(t) CELL STATE update o(t) = σ( W_o · z(t) + b_o ) OUTPUT gate h(t) = o(t) ⊙ tanh( C(t) ) HIDDEN STATE output
10 — VARIANTS

Variants — GRU, BiLSTM, Stacked LSTM

How LSTM evolved and what architectures are used in practice today.

GRU — Gated Recurrent Unit (Cho et al., 2014)

Simplifies LSTM by merging cell state and hidden state, and replacing the forget + input gates with a single update gate.

z(t) = σ( W_z · [h(t-1); x(t)] + b_z ) Update gate r(t) = σ( W_r · [h(t-1); x(t)] + b_r ) Reset gate h̃(t) = tanh( W_h · [r(t)⊙h(t-1); x(t)] + b_h ) Candidate h(t) = (1 − z(t)) ⊙ h(t-1) + z(t) ⊙ h̃(t) Final hidden state

Bidirectional LSTM

Runs two LSTMs simultaneously — one forward (t=1 to T) and one backward (t=T to 1). Their hidden states are concatenated, giving every position access to both past and future context.

h(t) = [ h_forward(t) ; h_backward(t) ] Use when: full sequence is available at inference time. Best for: sentiment analysis, NER, machine translation encoding. Cannot be used for: streaming / online prediction.

11 — APPLICATIONS

Real-world applications

Where RNN and LSTM are deployed, and what architecture fits each task.

🌐
Machine Translation
Encoder-decoder LSTM. Encoder reads source sentence into fixed-size context vector. Decoder generates target language word by word. Google Translate used 8-layer LSTM before switching to Transformers in 2016.
🎤
Speech Recognition
Bidirectional LSTM over Mel spectrogram frames. Still widely used in production systems. Apple Siri, Google Assistant, and Amazon Alexa all use LSTM-based acoustic models.
✍️
Text Generation
Character-level or word-level LSTM predicting the next token. Can generate coherent text, code, music notation. Predecessor to modern GPT-style transformers.
😊
Sentiment Analysis
BiLSTM over word embeddings, then classification head. Captures both forward context (what came before) and backward context (what comes after each word).
📈
Time-Series Forecasting
LSTM over historical values, forecasting future prices, demand, sensor readings. Still dominant in many industrial forecasting applications where Transformers haven't displaced it.
🏷️
Named Entity Recognition
BiLSTM-CRF model. BiLSTM encodes sentence context. CRF layer ensures globally consistent label sequences. State of the art before BERT (2018).
12 — SUMMARY

Summary & the architecture lineage

Everything in one place, and where the field went next.

The complete mental model

PROBLEM: Sequential data requires context from prior steps. Standard ANNs have no memory — each input processed in isolation. SOLUTION: RNN — add a hidden state h(t) that carries memory through time. h(t) = tanh( W·x(t) + U·h(t-1) + b ) Same W, U, b at every step. Weight identity across time. PROBLEM: Gradient must flow back T steps via repeated multiplication. tanh'(z) × W < 1 at each step. Product → 0 exponentially. Early steps get zero gradient signal. Vanishing gradient. SOLUTION: LSTM — separate long-term memory C(t) from short-term h(t). Cell state updated via addition: C(t) = f⊙C(t-1) + i⊙g Gradient flows back through addition, not tanh chains. Learned forget gate f(t) ≈ 1 keeps gradient intact. EVOLUTION: GRU (2014) — fewer parameters, similar performance. BiLSTM (2005) — process sequence in both directions. Attention (2015) — look directly at all past steps. Transformer (2017) — attend to everything simultaneously. No recurrence at all. Now the dominant paradigm.

What comes next — Transformers

LSTM solved vanishing gradients with an additive memory path. But it still compresses all history into a fixed-size vector C(t). Attention mechanisms (2015) allow the model to look directly at all previous hidden states, rather than a compressed summary. Transformers (2017) removed recurrence entirely — instead attending to all tokens simultaneously. This parallel processing made training dramatically faster and enabled scaling to billions of parameters, leading to GPT, BERT, and modern LLMs.
