00 — Introduction

What is a Convolutional Neural Network?

A CNN is a program that learns to recognize patterns in images — not by being told what to look for, but by seeing thousands of examples and figuring it out on its own.

When you look at a photo of a cat, your brain instantly processes edges, textures, shapes and assembles them into a coherent object. A computer, on the other hand, sees nothing but a grid of numbers. A Convolutional Neural Network (CNN) is the architecture that bridges that gap — it systematically transforms those raw numbers into meaningful visual understanding.

CNNs power the face unlock on your phone, the object detection in self-driving cars, the cancer detection in medical imaging, and image search across the web. Understanding how they work is one of the most rewarding journeys in modern technology.

🎯 What you'll learn

By the end of this article you'll understand every layer of a CNN — what it does, why it exists, and the mathematics behind it — well enough to explain it to someone else.

The big picture: a CNN is an assembly line

Think of a CNN as an assembly line of inspectors. The first inspector looks for simple things — edges and colors. The next combines those into shapes. The one after that assembles shapes into object parts. The final inspector makes the decision: "that's a cat."

01 — Input

Images are Just Grids of Numbers

A computer has no concept of "color" or "brightness" as we experience them. Every image is stored as a matrix of integers from 0 to 255 — and that's exactly what the CNN receives.

Grayscale: one matrix

In a grayscale image, every pixel is a single number: 0 = pure black, 255 = pure white, everything in between is a shade of gray. A 28×28 grayscale image (like handwritten digits) is literally a 28×28 matrix containing 784 numbers.

Hover any pixel to see its brightness value

Figure 2. Left: the image as we see it visually. Right: the exact same image as a matrix of numbers the CNN processes.

Color images: three matrices stacked (RGB)

Color images have three separate layers — called channels: Red, Green, and Blue. Each channel is its own full matrix of 0–255 values describing how much of that color is present at each pixel. The CNN receives all three simultaneously, forming a 3D block called a tensor.

Three separate matrices → combined → full color result
Figure 3. A color image is three stacked matrices. The CNN processes all three channels together.
📐 The math

A grayscale 32×32 image = 32 × 32 × 1 = 1,024 numbers. A color 224×224 image (common in real CNNs) = 224 × 224 × 3 = 150,528 numbers. That's the raw input the CNN starts with.

02 — Convolution

The Convolution Operation

Convolution is the heart of a CNN. A small grid of numbers called a filter slides across the image, and at each position it performs a dot product — multiplying its values with the image patch beneath it and summing everything up.

The filter (kernel)

A filter is a small matrix — typically 3×3 — containing learned numbers. Different filter values detect different visual patterns. Here are five classic examples:

Filter values

Green = positive weight
Red = negative weight

What it detects

Figure 4. Five common filter patterns. In real CNNs, filter values aren't hand-crafted like these — they're learned automatically from training data.

The sliding dot product — step by step

The filter slides across the image one position at a time. At each position, it multiplies its 9 values against the 9 image pixels beneath it, then sums the products. This produces a single output number. Repeat for every position → one complete feature map.

Output[i, j] = Σ Σ Filter[r, s] × Image[i+r, j+s] + bias for r, s in {0,1,2} (the 3×3 filter positions)
Convolution formula for one output value at position (i, j)

Multiple filters → multiple feature maps

One convolution layer doesn't use just one filter — it uses many, typically 32 to 512. Each filter learns to detect something different. Every filter produces its own output grid (a feature map). Stack all those maps together and you get the layer's output: a 3D tensor.

🔑 Key insight

The filter values are not chosen by you. They start as random numbers and are gradually adjusted during training until each filter becomes a specialist detector for some useful visual pattern.

Output size formula

Output size = ⌊ (Input size + 2×Padding − Filter size) ÷ Stride ⌋ + 1 Example: 32×32 input, 3×3 filter, stride=1, padding=0 → ⌊(32-3)/1⌋+1 = 30×30

Stride controls how far the filter jumps each step (1 = maximum detail, 2 = halves output size). Padding adds a border of zeros around the image so the output stays the same size as the input — very common in practice.

03 — Activation

ReLU — The Simple Switch

After every convolution, the output passes through an activation function. The most popular choice — by far — is ReLU (Rectified Linear Unit). It does exactly one thing: replace every negative number with zero.

ReLU(x) = max(0, x) 5 → 5  |  12 → 12  |  −3 → 0  |  −8 → 0

Before ReLU

→

After ReLU: max(0, x)

Red cells = negative (killed). Green cells = positive (kept).

Figure 6. ReLU removes all negative activations. Negatives usually mean "feature not found here" — keeping them would muddy the network's signal.

Why does this matter so much?

Without an activation function, stacking many layers of matrix multiplication would be mathematically equivalent to a single layer. ReLU introduces non-linearity — the ability to learn curved decision boundaries rather than just straight lines. It also makes training faster because its gradient is just 1 (for positive inputs) — no vanishing gradient problem.

04 — Pooling

Max Pooling — Compress and Generalize

After convolution + ReLU, the feature maps are still large. Max pooling shrinks them by scanning a small window and keeping only the maximum value in each region — discarding exact position but preserving signal strength.

How it works

A 2×2 max pool with stride 2 divides the feature map into non-overlapping 2×2 blocks and replaces each block with its single largest value. A 4×4 map becomes 2×2 — four times fewer numbers.

Output 2×2 (max of each region)

Colors show the four 2×2 regions. The winner (maximum) from each region survives.

Figure 7. 2×2 max pooling reduces 16 numbers to 4, keeping only the strongest activation from each neighborhood.
Output[i, j] = max( X[2i, 2j], X[2i, 2j+1], X[2i+1, 2j], X[2i+1, 2j+1] )
Max pooling with pool size 2×2, stride 2. Zero learnable parameters.

Four reasons pooling exists

01
Translation invariance
A feature present anywhere in the 2×2 window still produces the same max. Exact pixel position stops mattering.
02
Dimensionality reduction
4× fewer numbers per layer. Every subsequent layer is 4× cheaper to compute and store.
03
Grows receptive field
After pooling, the next conv layer "sees" a larger region of the original image — enabling big-picture understanding.
04
Reduces overfitting
Fewer numbers in later layers = fewer weights to learn = the network generalizes instead of memorizing.
05 — Depth

Stacking Layers Builds Complexity

Each convolution layer doesn't see the original image — it sees the output of the previous layer. This means later layers are automatically building on simpler features to detect more complex ones.

Layer 2's filters have shape 3×3×32 (not 3×3×3 like layer 1) — they reach through all 32 input feature maps at once. This lets them ask: "Is there a vertical edge here AND a horizontal edge there, at this relative position?" — which is exactly what a corner is.

💡 The combination principle

Each deeper layer's filters combine the previous layer's detectors in new spatial arrangements. Corner = vertical edge + horizontal edge meeting at a point. Curve = a sequence of edges arranged in an arc. Eye = curve above + circle + texture. The CNN discovers all of this automatically.

Feature hierarchy in a trained CNN

Figure 8. What real trained CNNs actually learn at each depth. No layer is told what to detect — the hierarchy emerges from training alone.

The receptive field grows with depth

Each neuron in layer 1 "sees" a 3×3 patch of the original image. But a neuron in layer 2 connects to a 3×3 patch of layer 1, and each of those neurons saw a 3×3 region — so layer 2 effectively sees a 5×5 patch of the original. By layer 5, a single neuron can respond to activity across a large portion of the entire image.

Receptive field at layer L = 1 + L × (filter size − 1) With 3×3 filters: Layer 1 = 3×3 | Layer 2 = 5×5 | Layer 3 = 7×7 | ... With 2×2 max pooling added, the receptive field doubles at each pooling step.

06 — Classification

The Fully Connected Layer

After the convolutional backbone extracts features, it hands off to a fundamentally different kind of layer — one where every single extracted feature potentially influences every output decision.

Step 1: Flatten

The 3D feature volume (height × width × depth) is "unrolled" into a single long 1D vector. No math — just reshaping. A 7×7×512 output becomes a vector of 25,088 numbers. This is the input to the first FC layer.

Step 2: The fully connected calculation

Every input neuron connects to every output neuron through a learned weight. Each output neuron computes a weighted sum of all inputs, adds its bias, and applies ReLU:

output_j = ReLU( Σᵢ wᵢⱼ × xᵢ + bⱼ ) where xᵢ = each input value, wᵢⱼ = weight connecting input i to output j, bⱼ = bias for output j

A fully connected layer with 25,088 inputs and 4,096 outputs has 25,088 × 4,096 + 4,096 ≈ 102 million parameters. This is why FC layers dominate the total parameter count of classic CNNs like VGG.

Step 3: Softmax — turning scores into probabilities

The final FC layer produces one raw score (called a logit) per class. Softmax converts these into probabilities that all add up to 100%:

P(class i) = e^(score_i) ÷ Σⱼ e^(score_j)
The exponential (eˣ) amplifies differences — the leading class dominates the probability.

Example: raw logit scores from the last FC layer → softmax probabilities

Figure 10. Softmax always produces values between 0 and 1 that sum to exactly 1. A score of 4.2 for "cat" vs 1.1 for "dog" gives a much larger probability difference than a ratio of 4.2/1.1 would suggest — because of the exponential.
07 — Training

How the CNN Learns

Every weight — every single number in every filter, every FC connection — starts as a random value and gradually improves through a feedback loop repeated hundreds of thousands of times.

01
Forward pass

The image flows forward through every layer — conv, ReLU, pool, FC — producing a final prediction. All intermediate activations are saved for step 3.

02
Compute the loss

Compare the prediction to the correct label using a loss function. For classification: Loss = −log(P(correct class)). A confident correct answer → small loss. A confident wrong answer → huge loss.

03
Backpropagation

Using the chain rule of calculus, compute the gradient ∂L/∂W for every weight — "if I nudge this weight up, does the loss go up or down?" This flows backward from the loss, through the FC layers, through pooling, through conv layers, all the way to the input.

04
Gradient descent update

Nudge every weight in the direction that reduces the loss:

W_new = W_old − α × ∂L/∂W α (learning rate) controls step size — too large = overshooting, too small = very slow convergence
Gradient descent — the core of all neural network learning
05
Repeat for every training example

Show the next image, repeat steps 1–4. After seeing millions of labeled images, each filter's weights settle into patterns that reliably detect useful visual features. The filter that started as random noise becomes a sharp edge detector, a curve finder, an eye recognizer — purely by reducing the loss.

Why this is remarkable

Nobody told the CNN that edges are useful. Nobody told it that corners are made of edges, or that eyes are made of corners. It discovered the entire feature hierarchy on its own, purely from the pressure of getting predictions right.

When you visualize trained CNN filters, they look exactly like edge detectors, Gabor filters, and blob detectors that computer vision researchers spent decades designing by hand — the CNN rediscovered all of them from first principles.

08 — Summary

The Complete Picture

Let's trace one image all the way through a CNN, end to end:

Figure 11. Complete CNN pipeline with shapes and roles at each stage.

Every operation in one place

1
Input image

Stored as H × W × C numbers (0–255 per channel). Divide by 255 to normalize to 0–1 range before feeding in.

2
Convolution

N filters of shape K×K×C_in slide across the input. Each produces one feature map. Output: H′ × W′ × N. Parameters: (K×K×C_in + 1) × N.

3
ReLU activation

max(0, x) applied element-wise. No parameters. Introduces non-linearity so the network can learn complex patterns.

4
Max pooling

Slide a 2×2 window, keep the maximum. Halves spatial dimensions. Adds translation invariance. No parameters.

5
Repeat Conv + ReLU + Pool

Each new conv layer detects combinations of the previous layer's features. Complexity compounds: edges → corners → shapes → parts → objects.

6
Flatten

Unroll the 3D feature volume into a 1D vector. No math — just reshaping. This connects the feature extractor to the classifier.

7
Fully connected layers

Every feature connects to every output. Learns which combinations of extracted features correspond to each class. Output_j = ReLU(Σ wᵢⱼ xᵢ + bⱼ).

8
Softmax output

Converts raw scores into probabilities summing to 100%. The class with the highest probability is the prediction.

🏆 The key insight to remember

A CNN is fundamentally a hierarchy of learned pattern detectors. Each layer's filters combine the previous layer's outputs in new spatial arrangements, building up from simple edges to complex objects — all discovered automatically from labeled examples and gradient descent. No human ever needs to define what an edge is, what a corner is, or what a face is. The network figures it all out on its own.

You now understand CNNs — from pixels to predictions.

Every layer has a clear job. Every operation has clean mathematics. And the whole system learns by itself, simply by trying to reduce a number that measures how wrong it was.

The same principles explored here power the most sophisticated visual AI systems in the world — scaled up with more layers, more filters, skip connections, and billions of training examples. But the core idea? It's exactly what you just read.
